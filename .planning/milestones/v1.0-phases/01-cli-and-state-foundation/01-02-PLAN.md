  <name>Task 2: branchos init command</name>
  <files>src/cli/init.ts, src/state/config.ts, tests/cli/init.test.ts</files>
  <behavior>
    - Running init in a git repo creates .branchos/shared/, .branchos/workstreams/, .branchos/config.json
    - config.json contains { schemaVersion: 1 }
    - .branchos-runtime/ is added to .gitignore
    - Init auto-commits with message "chore: initialize branchos"
    - Running init twice does not error and does not create duplicate commits
    - Running init outside a git repo prints error "Not a git repository. Run `git init` first."
    - Init reports what was created vs what already existed
    - --json flag outputs JSON instead of colored text
  </behavior>
  <action>
    1. Create `src/state/config.ts`:
       - Import CURRENT_SCHEMA_VERSION from './schema.js'
       - Export interface BranchosConfig { schemaVersion: number }
       - Export function createDefaultConfig(): BranchosConfig - returns { schemaVersion: CURRENT_SCHEMA_VERSION }

    2. Create `src/cli/init.ts`:
       - Import { Command } from 'commander'
       - Import GitOps, constants, output utilities, config creator
       - Export function registerInitCommand(program: Command): void
       - Command: 'init', description: 'Initialize BranchOS in this repository'
       - Option: --json for machine-readable output
       - Action handler:
         a. Create GitOps instance
         b. Check isGitRepo() - if false, call error() and process.exit(1)
         c. Get repo root via getRepoRoot()
         d. Track created vs skipped items
         e. Create directories if not exist: .branchos/shared/, .branchos/workstreams/
         f. Create .branchos/config.json if not exist with createDefaultConfig()
         g. Add '.branchos-runtime/' to .gitignore if not already present (read file, check, append if missing, create if no .gitignore)
         h. If any new items were created:
            - git add .branchos/ and .gitignore
            - git commit with message "chore: initialize branchos"
            - Report what was created
         i. If nothing new: report "Already initialized, nothing to do"
       - Use fs/promises for all file I/O (async, not sync)

    3. Wire into CLI: ensure registerInitCommand is called in src/cli/index.ts (should already be from Task 1)

    4. Write tests FIRST:
       - tests/cli/init.test.ts:
         - Setup: create temp dir, git init, configure git user.name and user.email for commit
         - Test: init creates .branchos/shared/, .branchos/workstreams/, .branchos/config.json
         - Test: config.json has schemaVersion: 1
         - Test: .gitignore contains .branchos-runtime/
         - Test: git log shows "chore: initialize branchos" commit
         - Test: running init twice does not error, second run creates no new commit
         - Test: init outside git repo exits with error message
         - Use child_process.execSync or direct function calls to test

    5. Build and test end-to-end: `npm run build && node dist/index.cjs init` in a test repo.
