    - isProtectedBranch("feature/foo") returns false

       - Export async function writeMeta(filePath: string, meta: WorkstreamMeta): Promise<void>

       - Export function createInitialState(): WorkstreamState - returns scaffold
       - Export async function readState(filePath: string): Promise<WorkstreamState>
       - Export async function writeState(filePath: string, state: WorkstreamState): Promise<void>

  <behavior>

    - createWorkstream with nameOverride="custom-name" uses "custom-name" as ID instead of slug
    - createWorkstream on "main" branch throws error about protected branches
    - createWorkstream when slug already exists throws error suggesting --name
    - branchos workstream create (CLI) on a valid branch creates workstream and prints success
    - branchos workstream create --name custom-id uses override
    - branchos workstream create --json outputs JSON
    - branchos workstream create on detached HEAD errors clearly
    - branchos workstream create in uninitialized repo (no .branchos/) errors with suggestion to run init

  <action>
    1. Create `src/workstream/create.ts`:
       - Import GitOps, slugifyBranch, isProtectedBranch, createMeta, createInitialState, discoverWorkstreams, writeMeta, writeState, constants
       - Export interface CreateWorkstreamResult { workstreamId: string; branch: string; path: string; created: boolean }
       - Export async function createWorkstream(options: { repoRoot: string; nameOverride?: string }): Promise<CreateWorkstreamResult>
         a. Create GitOps instance
         b. Get current branch
         c. Check isProtectedBranch - if true, throw Error("Cannot create workstream on protected branch '<branch>'. Switch to a feature branch first.")
         d. Derive workstreamId: if nameOverride, validate and use it; else slugifyBranch(branch)
         e. Validate workstreamId is not empty after slugification
         f. Check .branchos/ exists - if not, throw Error("BranchOS not initialized. Run `branchos init` first.")
         g. Check for collision: discoverWorkstreams, if workstreamId already exists, throw Error("Workstream '<id>' already exists. Use --name <custom-name> to specify a different ID.")
         h. Create directory: .branchos/workstreams/<workstreamId>/
         i. Write meta.json via writeMeta(createMeta(workstreamId, branch))
         j. Write state.json via writeState(createInitialState())
         k. Git add and commit: "chore: create workstream <workstreamId>"
         l. Return result object

       - Export function registerWorkstreamCommands(program: Command): void
       - Create workstream command group
       - Add 'create' subcommand:
         - Option: --name <name> "Override auto-derived workstream ID"
         - Option: --json "Output as JSON"
         - Action handler:
           a. Create GitOps, check isGitRepo
           b. Get repoRoot
           c. Call createWorkstream({ repoRoot, nameOverride: options.name })
           d. Output result with success message
           e. Catch errors and output with error()
