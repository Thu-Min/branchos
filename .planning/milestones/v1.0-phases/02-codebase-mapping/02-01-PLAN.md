       ```
       description: Generate or refresh the codebase map in .branchos/shared/codebase/
       allowed-tools: Read, Glob, Grep, Write, Bash(git rev-parse HEAD), Bash(git add *), Bash(git commit *)
       ---
       ```

    3. The prompt body should instruct Claude to perform these steps in order:

       **Step 1: Read configuration**
       - Read `.branchos/config.json`
       - If `map.excludes` does not exist, update config.json to add default excludes: ["node_modules", "dist", "build", ".branchos", ".git", "*.lock", "*.min.*"]
       - If `map.stalenessThreshold` does not exist, set it to 20

       **Step 2: Get current commit hash**
       - Run `git rev-parse HEAD` to get the current commit hash

       **Step 3: Analyze the codebase**
       - Use Glob and Read tools to explore the repository, excluding paths matching the exclude patterns
       - For each of the 5 map files, provide explicit instructions on what to analyze and what format to use:

       **ARCHITECTURE.md**: High-level structure, entry points, data flow between modules. Include a directory tree of the top 2 levels. Describe how data flows through the system.

       **MODULES.md**: Directory-level summaries (NOT file-level inventory). For each major directory: purpose, key exports, relationships to other modules. Only call out individual files when they are important entry points or contain complex logic.

       **CONVENTIONS.md**: Code patterns ONLY (not workflow). Cover: naming conventions, file organization patterns, state management approach, error handling patterns, import/export patterns. Include brief code examples where helpful.

       **STACK.md**: List each dependency from package.json with its purpose/role. Do NOT pin versions (package.json tracks versions). Group by: runtime dependencies, dev dependencies.

       **CONCERNS.md**: Descriptive observations of tech debt, complexity hotspots, potential risks. Do NOT include prescriptive fix suggestions. Just describe what you observe.

       **Step 4: Write map files**
       - Write each file to `.branchos/shared/codebase/` with this metadata header:
         ```
         ---
         generated: <ISO 8601 timestamp>
         commit: <HEAD hash from step 2>
         generator: branchos/map-codebase
         ---
         ```
       - Create the directory if it does not exist

       **Step 5: Auto-commit**
       - Run: `git add .branchos/shared/codebase/ARCHITECTURE.md .branchos/shared/codebase/MODULES.md .branchos/shared/codebase/CONVENTIONS.md .branchos/shared/codebase/STACK.md .branchos/shared/codebase/CONCERNS.md .branchos/config.json`
       - Run: `git commit -m "chore(branchos): refresh codebase map"`

       Include `$ARGUMENTS` at the end so users can pass optional focus areas.
