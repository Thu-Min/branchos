<task type="auto" tdd="true">
  <name>Task 1: GitOps extensions and staleness detection module</name>
  <files>src/git/index.ts, src/map/staleness.ts, src/map/index.ts, tests/git/index.test.ts, tests/map/staleness.test.ts</files>
  <behavior>
    - getHeadHash: returns current HEAD commit hash as trimmed string
    - getCommitsBehind: given a valid ancestor hash, returns count of commits between that hash and HEAD
    - getCommitsBehind: given a hash that does not exist in history (rebase scenario), returns -1 (not throw)
    - getCommitsBehind: given HEAD hash itself, returns 0
    - checkStaleness: when no map files exist, returns { exists: false, commitsBehind: -1, isStale: false }
    - checkStaleness: when map files exist and are current (0 commits behind), returns { exists: true, commitsBehind: 0, isStale: false }
    - checkStaleness: when map is N commits behind (N < threshold), returns { exists: true, commitsBehind: N, isStale: false }
    - checkStaleness: when map is N commits behind (N >= threshold), returns { exists: true, commitsBehind: N, isStale: true }
    - checkStaleness: when map commit hash is unknown (-1 from getCommitsBehind), returns { exists: true, commitsBehind: -1, isStale: true }
  </behavior>
  <action>
    1. Add two methods to GitOps class in src/git/index.ts:

       `async getHeadHash(): Promise<string>` - uses `this.git.revparse(['HEAD'])`, trims result.

       `async getCommitsBehind(fromHash: string): Promise<number>` - uses `this.git.raw(['rev-list', '--count', fromHash + '..HEAD'])`. Wraps in try/catch: on error returns -1 (handles unknown hash after rebase). Parses result with parseInt(result.trim(), 10).

    2. Create src/map/staleness.ts with:
       - `StalenessResult` interface: { exists: boolean; commitsBehind: number; isStale: boolean; mapCommit?: string; headCommit?: string; generated?: string }
       - `checkStaleness(repoRoot: string, threshold?: number): Promise<StalenessResult>` function:
         - Constructs codebase dir path: join(repoRoot, BRANCHOS_DIR, SHARED_DIR, CODEBASE_DIR)
         - Creates GitOps with repoRoot as cwd
         - Iterates MAP_FILES, tries to read each file (readFile from fs/promises), parse metadata with parseMapMetadata
         - On first successful metadata parse: gets headHash, gets commitsBehind, returns full result
         - If no files found or no valid metadata: returns { exists: false, commitsBehind: -1, isStale: false }
         - Default threshold is 20
         - isStale = behind === -1 || behind >= threshold

    3. Update src/map/index.ts barrel to also export StalenessResult and checkStaleness from staleness.ts.

    4. Add tests to tests/git/index.test.ts for getHeadHash and getCommitsBehind (use temp git repos with multiple commits, and test with nonexistent hash).

    5. Create tests/map/staleness.test.ts:
       - Test with no codebase directory (returns exists: false)
       - Test with map files that have valid metadata matching HEAD (returns exists: true, commitsBehind: 0, isStale: false)
       - Test with map files that have old commit hash (create commits after writing map files to get a non-zero count)
       - Test staleness threshold (set threshold to 1, verify isStale becomes true at 1 commit behind)
       - For these tests: create temp git repos, write actual map files with metadata headers, make commits, then call checkStaleness
  </action>
  <verify>
    <automated>npx vitest run tests/git/index.test.ts tests/map/staleness.test.ts --reporter=verbose</automated>
  </verify>
  <done>GitOps has getHeadHash() and getCommitsBehind() methods. checkStaleness reads map file metadata, compares against HEAD, reports staleness with configurable threshold. Handles missing maps and unknown hashes gracefully. All tests pass.</done>
