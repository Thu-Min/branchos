Create three slash command files following the established map-codebase.md pattern. Each uses YAML frontmatter with `description` and `allowed-tools`. Each instructs Claude Code to:
- Detect current branch via `git branch --show-current`

- Read workstream's `state.json`, check for existing phases
- Generate the artifact, write it, update state.json, auto-commit

**1. `.claude/commands/discuss-phase.md`**

```
---
description: Create or update discussion context for current workstream phase
allowed-tools: Read, Glob, Grep, Write, Bash(git *), Bash(npx branchos *)
---
```

Prompt instructions:
- Determine target phase: if `$ARGUMENTS` contains a number, use that phase. Otherwise auto-increment (phases.length + 1 or currentPhase if already active).
- If discuss.md already exists for the target phase, warn user and ask before overwriting.
- Read the codebase map from `.branchos/shared/codebase/` for architecture context.
- Generate discuss.md with sections: `## Goal`, `## Requirements`, `## Assumptions`, `## Unknowns`, `## Decisions`. Use user's arguments as guidance for what to discuss.
- Write to `.branchos/workstreams/<id>/phases/<n>/discuss.md` (create directory if needed).
- Update state.json: if phase doesn't exist in phases array, add it. Set discuss step to `{ status: 'complete', createdAt: <ISO timestamp>, updatedAt: <ISO timestamp> }`. Set currentPhase to this phase number.
- If any decisions were identified, also append them to `.branchos/workstreams/<id>/decisions.md` using the format: `### <title>`, `**Phase:** <n>`, `**Context:** <why>`, `**Decision:** <choice>`, `**Alternatives considered:** <list>`, `---`.
- Auto-commit all changed files with message `chore(branchos): discuss phase <n> for <workstream-id>`

**2. `.claude/commands/plan-phase.md`**

```
---
description: Create implementation plan for current workstream phase
allowed-tools: Read, Glob, Grep, Write, Bash(git *), Bash(npx branchos *)
---
```

Prompt instructions:
- Determine target phase (same logic as discuss-phase).
- Warn if no discuss.md exists for this phase: "No discuss.md found for phase N. Planning without discussion context."
- Read discuss.md for this phase if it exists. Read codebase map. Read decisions.md.
- Generate plan.md with sections: `## Objective`, `## Tasks` (each task has: description, `### Affected Files` with backtick-quoted file paths as list items, dependencies, risks). Also include a top-level `## Affected Files` section consolidating all unique files across tasks.
- CRITICAL: The `### Affected Files` format MUST use `- \`path/to/file\`` syntax per line. This is parsed by drift detection.
- Write to `.branchos/workstreams/<id>/phases/<n>/plan.md`.
- Update state.json: set plan step to `{ status: 'complete', createdAt, updatedAt }`. Store `planBaseline` on the phase object by running `git rev-parse HEAD`.
- If decisions were made during planning, append to decisions.md.
- Auto-commit with message `chore(branchos): plan phase <n> for <workstream-id>`

**3. `.claude/commands/execute-phase.md`**

```
---
description: Update execution state for current workstream phase
allowed-tools: Read, Glob, Grep, Write, Bash(git *), Bash(npx branchos *)
---
```

Prompt instructions:
- Determine target phase.
- Warn if no plan.md exists for this phase.
- Read plan.md to understand the tasks. Read existing execute.md if present (update, not overwrite).
- Generate/update execute.md with sections: `## Completed Tasks`, `## In Progress`, `## Remaining`, `## Blockers`. For each task from plan.md, assess status based on git history and file state.
- Write to `.branchos/workstreams/<id>/phases/<n>/execute.md`.
- Update state.json: set execute step status. If all tasks are complete, set phase status to 'completed'.
- Auto-commit with message `chore(branchos): execute phase <n> for <workstream-id>`

All three slash commands end with `$ARGUMENTS` on the last line.
