  <behavior>
    - Missing PR-FAQ.md: returns { success: false, error: "No PR-FAQ.md found..." }
    - Missing .branchos/ directory: returns { success: false, error: "BranchOS not initialized..." }
    - First ingestion of valid PR-FAQ: copies to .branchos/shared/PR-FAQ.md, writes prfaq-meta.json, auto-commits, returns action: 'ingested'
    - Re-ingestion of unchanged PR-FAQ: returns action: 'unchanged', no commit
    - Re-ingestion of modified PR-FAQ: updates stored copy and metadata, auto-commits, returns action: 'updated' with diff
    - Non-PR-FAQ document (fewer than 2 sections) without --force: calls promptYesNo, aborts on 'no'
    - Non-PR-FAQ document with --force: proceeds without prompting
    - --json flag: returns result object without console output
    - Success output includes section count, missing section warnings, and next-step suggestion
    - Result includes sectionsFound and sectionsMissing arrays
  </behavior>
  <action>
    Create `src/cli/ingest-prfaq.ts` following the `init.ts` pattern exactly:

    1. `ingestPrfaqHandler(options: IngestPrfaqOptions): Promise<IngestPrfaqResult>`:
       - Resolve cwd via `options.cwd || process.cwd()`
       - Create `GitOps(cwd)`, verify git repo, get repo root
       - Check `.branchos/` exists (error if not: "BranchOS not initialized. Run `branchos init` first.")
       - Check `PR-FAQ.md` at repo root exists (error if not: "No PR-FAQ.md found in repo root. Create a PR-FAQ document at ./PR-FAQ.md and try again.")
       - Read PR-FAQ.md content ONCE into memory
       - Call `detectSections(content)` to get found/missing
       - If `!isLikelyPrfaq(content)` and `!options.force`: call `promptYesNo("This doesn't look like a PR-FAQ (only N/8 expected sections found). Continue anyway? (y/N) ")`. If false, return `{ success: false, action: 'unchanged', error: 'Aborted by user', ... }`
       - Call `hashContent(content)` to get new hash
       - Read existing metadata via `readMeta(sharedDir)`
       - If metadata exists and hash matches: return `{ success: true, action: 'unchanged', ... }` with human output "No changes detected."
       - If metadata exists and hash differs: compute `diffSections(oldContent, content)`, set action to 'updated'
       - If no metadata: set action to 'ingested'
       - Write content to `.branchos/shared/PR-FAQ.md`
       - Write metadata via `writeMeta(sharedDir, { contentHash, ingestedAt: new Date().toISOString(), version: 1, sectionsFound, sectionsMissing, sourceSize: Buffer.byteLength(content) })`
       - Auto-commit via `git.addAndCommit(['.branchos/shared/PR-FAQ.md', '.branchos/shared/prfaq-meta.json'], commitMsg)`. Commit message: "chore: ingest PR-FAQ" for first ingestion, "chore: update PR-FAQ" for re-ingestion.
       - Human output (when not --json): `success("Ingested PR-FAQ (N/8 sections found).")` or `success("Updated PR-FAQ.")`. If sectionsMissing.length > 0: `warning("Missing sections: ...")`. Always: `output("Next: run /branchos:plan-roadmap to generate your roadmap")`.
       - If action is 'updated' and diff has changes: output summary like "Updated: Problem, Solution. Added: Internal FAQ."
       - Return IngestPrfaqResult

    2. `registerIngestPrfaqCommand(program: Command): void`:
       - `.command('ingest-prfaq')`
       - `.description('Ingest PR-FAQ document for project planning')`
       - `.option('--json', 'Output in JSON format', false)`
       - `.option('--force', 'Skip confirmation prompts', false)`
       - `.action(async (opts) => { const result = await ingestPrfaqHandler({ json: opts.json, force: opts.force }); if (!result.success) process.exit(1); })`

    3. Write tests FIRST in `tests/cli/ingest-prfaq.test.ts`. Use a temp directory for each test (fs.mkdtemp). Mock `GitOps` (vi.mock) to avoid real git operations -- mock `isGitRepo` to return true, `getRepoRoot` to return temp dir, `addAndCommit` to be a no-op spy. Mock `promptYesNo` where needed. Create real files (PR-FAQ.md, .branchos/shared/) in temp dir for each test.
