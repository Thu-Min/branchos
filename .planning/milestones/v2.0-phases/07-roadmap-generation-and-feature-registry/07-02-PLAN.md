    Create `src/cli/plan-roadmap.ts` following the `ingestPrfaqHandler` pattern exactly:

    Define `PlanRoadmapOptions` interface: `{ json: boolean; force: boolean; cwd?: string; roadmapData?: RoadmapData }`.

    The `roadmapData` field is the key design decision: the slash command (Claude) reads the PR-FAQ, infers milestones and features, and passes structured `RoadmapData` to the handler. The CLI command itself does NOT do AI inference -- it validates, writes, and commits. When called via CLI directly (without roadmapData), the handler reads `PR-FAQ.md` and outputs it to stdout with instructions for the user to provide structured data. But the primary flow is via slash command where roadmapData is pre-populated.

    For testability, the handler accepts `roadmapData` directly. The handler flow:
    1. Validate preconditions: git repo, .branchos exists, PR-FAQ ingested (readMeta check)
    2. Check if ROADMAP.md exists -- error without --force
    3. If --force and features/ exists, remove all existing feature files (`rm` each `F-*.md`)
    4. Receive `roadmapData` (from slash command) or error with guidance to use slash command
    5. Generate ROADMAP.md content via `generateRoadmapMarkdown(roadmapData)`
    6. Write ROADMAP.md to `.branchos/shared/ROADMAP.md`
    7. Write each feature file via `writeFeatureFile`
    8. Auto-commit all written files via `git.addAndCommit`
    9. Print summary: milestone count, feature count, next step guidance
    10. Return `PlanRoadmapResult` with success, featureCount, milestoneCount

    Define `PlanRoadmapResult`: `{ success: boolean; featureCount: number; milestoneCount: number; error?: string }`.

    Tests: Mock GitOps (same pattern as `tests/cli/ingest-prfaq.test.ts`). Use temp directories. Test error cases (no PR-FAQ, existing roadmap without --force) and success case (provides roadmapData, verifies files written and committed). Do NOT test AI inference -- that happens in the slash command.

    ```typescript
    export function registerPlanRoadmapCommand(program: Command): void {
      program
        .command('plan-roadmap')
        .description('Generate roadmap and features from ingested PR-FAQ')
        .option('--json', 'Output in JSON format', false)
        .option('--force', 'Overwrite existing roadmap', false)
        .action(async (opts) => {
          const result = await planRoadmapHandler({ json: opts.json, force: opts.force });
          if (!result.success) process.exit(1);
        });
    }
    ```
