1. Create src/phase/index.ts with these exported functions:

- `ensurePhaseDir(repoRoot, workstreamId, phaseNumber)`: Creates `.branchos/workstreams/<id>/phases/<n>/` directory using `mkdir({ recursive: true })`. Returns the path.

- `createPhase(state)`: Takes a WorkstreamState, returns a new WorkstreamState with a new Phase appended. New phase number = `state.phases.length + 1`. New phase has `status: 'active'`, all steps `status: 'not-started'`. Sets `currentPhase` to the new phase number. Does NOT mutate input (return new object, consistent with shallow copy pattern from schema.ts).

- `getCurrentPhase(state)`: Returns the Phase object where `phase.number === state.currentPhase`, or null if `currentPhase === 0`.

- `updatePhaseStep(state, phaseNumber, step, updates)`: Updates a specific step (discuss/plan/execute) on a specific phase. `step` is `'discuss' | 'plan' | 'execute'`. `updates` is `Partial<PhaseStep>`. If step is 'plan' and updates include status 'complete' or 'in-progress', accept optional `planBaseline` in updates to store on the Phase. Returns new state (immutable).

- `resolveCurrentWorkstream(repoRoot)`: Uses GitOps to get current branch, reads workstreams dir, scans meta.json files to find matching branch. Returns `{ id, path }` or null. Import from `src/workstream/discover.ts` and use `readFile` for meta.json.

2. Create src/phase/decisions.ts with these exported functions:

- `DecisionEntry` interface: `{ title, phase, context, choice, alternatives: string[] }`

- `formatDecisionEntry(entry)`: Returns formatted markdown string with `### title`, `**Phase:**`, `**Context:**`, `**Decision:**`, `**Alternatives considered:**` sections, ending with `---` separator.

- `readDecisions(filePath)`: Reads decisions.md, returns content string. Returns empty string if file does not exist (use try/catch on readFile).

- `appendDecision(filePath, entry)`: Reads existing content via readDecisions, appends formatted entry. If file is empty, adds a `# Decisions` header first. Writes full content back.
