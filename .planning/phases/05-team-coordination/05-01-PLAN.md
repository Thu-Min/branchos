  <files>src/git/index.ts, src/workstream/status.ts, src/cli/status.ts, tests/workstream/status.test.ts, tests/cli/status.test.ts</files>
  <behavior>
    - GitOps.getChangedFilesForBranch(branch, baseBranch): returns file list from merge-base diff; returns empty array on error
    - GitOps.isBranchMerged(branch, into): returns true if branch is ancestor of target; returns false on error
    - statusHandler({ all, json }): gathers all workstreams with meta + state, filters archived unless all=true
    - statusHandler returns array of StatusRow objects with id, branch, currentPhase (number + step), lastActivity, status
    - Status CLI prints table with columns: marker, Workstream, Branch, Phase, Last Activity, Status
    - Current branch workstream gets triangle marker; others get blank
    - --json outputs array of StatusRow objects
    - Handles empty workstreams directory gracefully (shows message)
    - Handles workstream with no phases (shows "No phases" in Phase column)
  </behavior>
  <action>
    1. Add two methods to GitOps class in src/git/index.ts:
       - `async getChangedFilesForBranch(branch: string, baseBranch: string): Promise<string[]>` -- runs `git diff --name-only $(git merge-base branch baseBranch)..branch`. Returns empty array on error.
       - `async isBranchMerged(branch: string, into: string): Promise<boolean>` -- runs `git merge-base --is-ancestor branch into`. Returns true if exit 0, false on error.

    2. Create src/workstream/status.ts with:
       - `StatusRow` interface: { id: string, branch: string, phase: string, lastActivity: string, status: string }
       - `StatusResult` interface: { rows: StatusRow[], currentBranch: string }
       - `statusHandler(options: { all?: boolean, json?: boolean, repoRoot?: string }): Promise<StatusResult | null>` -- discovers workstreams, reads meta + state for each, filters by status unless all=true, determines current branch for highlighting, outputs table or JSON.
       - Use `discoverWorkstreams()`, `readMeta()`, `readState()` from existing modules.
       - Phase column: `Phase N / step` where step is the first step with status 'in-progress', or 'discuss' if all are 'not-started'. If no phases: "No phases".
       - Last activity: meta.updatedAt formatted as relative time or ISO date.
       - Print using chalk + padEnd for column alignment (see RESEARCH.md code example).
       - If no workstreams found, print "No active workstreams." message.

    3. Create src/cli/status.ts following the registerXxxCommand pattern:
       - `registerStatusCommand(program: Command)`: registers `branchos status` with `--all` and `--json` flags.
       - Action calls statusHandler with options.

    4. Write tests in tests/workstream/status.test.ts and tests/cli/status.test.ts:
       - Test statusHandler with mocked discoverWorkstreams, readMeta, readState.
       - Test filtering of archived workstreams.
       - Test --all includes archived.
       - Test phase display logic (active phase/step, no phases case).
       - Test CLI registration (command name, options).

    - unarchiveHandler(workstreamId, options): reads meta, sets status to 'active', updates updatedAt, writes meta, commits

    - unarchiveHandler throws if workstream not found
    - unarchiveHandler throws if already active
