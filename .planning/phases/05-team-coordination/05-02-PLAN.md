<task type="auto" tdd="true">
  <name>Task 1: Conflict detection logic with tests</name>
  <files>src/workstream/conflicts.ts, tests/workstream/conflicts.test.ts</files>
  <behavior>
    - detectConflicts(workstreams: WorkstreamFiles[]): returns FileConflict[] sorted by severity (high first) then filename
    - FileConflict has file, severity ('high'|'medium'), and workstreams array with id and source ('planned'|'changed')
    - High severity: all entries for a file have source='changed'
    - Medium severity: at least one entry has source='planned'
    - Files touched by only one workstream are NOT conflicts
    - Deduplicates: if a workstream has same file in both planned and changed, prefer 'changed'
    - Empty inputs return empty array
    - gatherWorkstreamFiles reads plan.md for each phase to get planned files, uses GitOps for actual changed files
    - Missing plan.md treated as empty planned files (not an error)
    - Branches not available locally are skipped with warning
  </behavior>
  <action>
    1. Create src/workstream/conflicts.ts with:
       - `WorkstreamFiles` interface: `{ id: string, planned: string[], changed: string[] }`
       - `FileConflict` interface: `{ file: string, severity: 'high' | 'medium', workstreams: Array<{ id: string, source: 'planned' | 'changed' }> }`
       - `detectConflicts(workstreams: WorkstreamFiles[]): FileConflict[]` -- pure function. Build a Map<string, Array<{id, source}>>. For each workstream, add changed files with source='changed', then add planned files with source='planned' only if that workstream doesn't already have the file (changed takes priority). Filter to files touched by 2+ unique workstreams. Classify severity: if ALL entries for a file are 'changed' -> 'high', otherwise -> 'medium'. Sort by severity (high first) then filename.
       - `gatherWorkstreamFiles(options: { repoRoot: string, workstreamIds: string[], includeArchived?: boolean }): Promise<WorkstreamFiles[]>` -- for each workstream ID: read meta.json (skip if archived unless includeArchived), read state.json to find current phase, read plan.md from current phase directory via parseAffectedFiles for planned files, use GitOps.getChangedFilesForBranch(meta.branch, protectedBranch) for actual changed files (try each PROTECTED_BRANCHES until one works as base, fall back to empty array). Skip workstreams whose branches are not available locally (warn).
       - `detectConflictsHandler(options: { all?: boolean, json?: boolean, repoRoot?: string }): Promise<FileConflict[] | null>` -- resolves repo root, resolves current workstream, discovers all workstreams, gathers file data, runs detectConflicts. If `--all`: compare all workstreams. If default: filter to conflicts involving current workstream. Output conflicts grouped by file with severity labels using chalk (red for high, yellow for medium). If no conflicts, print "No file conflicts detected." If --json, output JSON array.

    2. Write tests in tests/workstream/conflicts.test.ts:
       - Test detectConflicts pure function:
         - Two workstreams with same changed file -> high severity
         - Two workstreams, one planned one changed same file -> medium
         - Two workstreams, both planned same file -> medium
         - No overlap -> empty array
         - Same workstream has file in both planned and changed -> dedup, prefer 'changed'
         - Three workstreams overlapping on same file -> all three listed
         - Sorted: high before medium, then alphabetical
       - Test gatherWorkstreamFiles with mocked file system and git (vi.mock pattern).
       - Test handler with mocked dependencies.
  </action>
  <verify>
    <automated>npx vitest run tests/workstream/conflicts.test.ts -x</automated>
  </verify>
  <done>detectConflicts correctly identifies file overlaps, classifies severity, and deduplicates. Handler gathers data from all workstreams and outputs results. Tests pass.</done>
