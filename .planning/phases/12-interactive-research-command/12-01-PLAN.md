    ```yaml
    ---
    ```

    **Body structure (natural language instructions for Claude, NOT code):**

    1. **Argument parsing (FIRST CHECK):** Parse `$ARGUMENTS` for flags. If `--save` is present, jump directly to the Save Flow. Otherwise treat the remaining text as the research topic.

    2. **Opening bookend -- Frame the research session:**
       - Identify the research topic from `$ARGUMENTS`
       - Load codebase context: read `.branchos/shared/codebase/ARCHITECTURE.md` if it exists for architecture awareness
       - Read existing research index: `!cat .branchos/shared/research/index.json 2>/dev/null || echo "[]"` to check for prior research on this topic
       - Present the research question back to the user with initial analysis

    3. **Interactive research (Claude drives with AskUserQuestion):**
       - Present structured research direction options as numbered choices. Example format:
         "I've identified these research areas for [topic]:
         1. [Area from codebase analysis]
         2. [Area from domain knowledge]
         3. [Area from architecture patterns]
         4. Other (describe what you'd like to explore)"
       - ALWAYS include an "Other" option for freeform follow-up (INT-02)
       - Use AskUserQuestion for each structured decision point
       - After the user responds, adapt the next question based on their answer (INT-03)
       - Use WebSearch and WebFetch for actual research during the conversation
       - Use Read, Glob, Grep to ground research in the codebase
       - Continue the research conversation until the user indicates they have enough information or wants to save
       - Guidelines for adaptive questioning: "Adapt your follow-up questions based on the user's responses. If they indicate interest in a specific area, drill deeper. If they seem satisfied with an area, move to the next. Do not follow a rigid script -- use the conversation context to determine the most useful next question."

    4. **Closing bookend -- Save flow (triggered by --save):**
       - Compile findings from the conversation into a research artifact
       - Ask the user to confirm/edit the summary before saving
       - The artifact MUST include a `## Summary` section as the first H2 (3-5 bullet points)
       - Followed by `## Findings` with detailed results
       - Use the Phase 11 API to persist:
         ```
         Read .branchos/shared/research/index.json to get existing IDs
         Generate next ID with nextResearchId
         Generate filename with researchFilename(id, topic)
         Write artifact with writeResearchFile to .branchos/shared/research/
         ```
       - Ask the user which features this research relates to (if any) for the `features` array
       - Git commit the new research file and updated index.json

    5. End with `$ARGUMENTS` placeholder (required by all BranchOS commands)
