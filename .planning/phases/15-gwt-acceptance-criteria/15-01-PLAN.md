    RED phase: Create `tests/roadmap/gwt-parser.test.ts` with failing tests for all behaviors above. Tests import from `src/roadmap/gwt-parser.ts`. Create a minimal stub file that exports the correct types and function signatures but returns empty results, so imports resolve but tests fail.

    Type definitions to create in `src/roadmap/gwt-parser.ts`:
    ```typescript
    export interface GwtStep {
      keyword: 'Given' | 'When' | 'Then';
      text: string;
    }
    export interface GwtBlock {
      id: string;         // "AC-1", "AC-2", etc.
      steps: GwtStep[];
    }
    export interface FreeformCriterion {
      text: string;       // Checklist item text without "- [ ] " prefix
    }
    export interface ParsedAcceptanceCriteria {
      gwtBlocks: GwtBlock[];
      freeformItems: FreeformCriterion[];
    }
    ```

    GREEN phase: Implement `parseAcceptanceCriteria(body: string): ParsedAcceptanceCriteria`:
    1. Find "## Acceptance Criteria" heading (case-sensitive, exact match at line start)
    2. Extract lines until next "## " heading or end of string
    3. Split into sections by "### AC-N" headings
    4. For each AC-N section: parse lines starting with Given/When/Then/And keywords
       - Track `lastKeyword` for And-continuation
       - If And appears without preceding keyword, mark block as invalid
    5. After parsing AC-N block: validate all three keywords (Given, When, Then) present
       - Valid: add as GwtBlock
       - Invalid: add each line as FreeformCriterion
    6. Collect loose "- [ ]" lines (outside AC-N blocks) as FreeformCriterion
    7. Ignore blank lines and non-matching lines within AC section

    Also implement `formatGwtChecklist(parsed: ParsedAcceptanceCriteria): string`:
    1. For each GwtBlock: render "- [ ] **{id}**" followed by indented steps
       - Steps that were originally "And" lines: render as "  - And {text}"
       - Other steps: render as "  - {keyword} {text}"
    2. For each FreeformCriterion: render "- [ ] {text}"
    3. Return joined with newlines, prefixed with "## Acceptance Criteria\n\n"

    To support And-rendering: track whether each step was originally an And line. Options:
    - Add an optional `wasAnd: boolean` field to GwtStep
    - Or store original keyword separately
    Use `wasAnd?: boolean` on GwtStep (simplest approach).
