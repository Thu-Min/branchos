       p. Call `createPr({title, body, baseBranch: defaultBranch, assignee})`
       q. Print success with PR URL

       Note: The confirmation flow (AskUserQuestion) happens in the slash command context, not in the CLI handler. The CLI handler just prints the body and proceeds. The slash command wraps it with confirmation. Actually, per CONTEXT.md: "Slash command displays assembled title + full PR body as text, AskUserQuestion with Create PR / Cancel". So the CLI should have a `--confirm` flag or the slash command should call the CLI in two steps. Simplest approach: CLI has `--dry-run` that just prints the body without creating, and the slash command does: (1) run `npx branchos create-pr --dry-run` to show body, (2) ask user, (3) run `npx branchos create-pr` to create. OR: the CLI always creates (no interactive stdin), and the slash command handles confirmation. Go with the second approach -- CLI is non-interactive, slash command wraps with confirmation.

       Revised: Add `--dry-run` flag. With `--dry-run`, print title + body and exit. Without, create the PR. Slash command calls `--dry-run` first, asks user, then calls without `--dry-run`.
