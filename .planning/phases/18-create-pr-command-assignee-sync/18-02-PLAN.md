    - findAssigneeForFeature only considers active workstreams (not archived)
    - findAssigneeForFeature returns null when workstreams directory does not exist
    - syncIssuesHandler calls gh issue edit --add-assignee after creating/updating issues
    - syncIssuesHandler skips assignee sync when findAssigneeForFeature returns null
    - syncIssuesHandler skips assignee sync in dry-run mode
    - syncIssuesHandler does not fail if --add-assignee call fails (warning only)
  </behavior>
  <action>
    Add to `src/cli/sync-issues.ts`:

    1. **Import readMeta** from `../state/meta.js` and `readdir` from `fs/promises` and `ghExec` from `../github/index.js` (ghExec may already be indirectly available -- check imports).

    2. **findAssigneeForFeature(repoRoot: string, featureId: string): Promise<string | null>**
       - Build path: `join(repoRoot, '.branchos', 'workstreams')`
       - `readdir(wsDir)` -- catch and return null if dir doesn't exist
       - Sort entries alphabetically for deterministic first-match
       - For each entry, read `join(wsDir, entry, 'meta.json')` via `readMeta`
       - If `meta.featureId === featureId && meta.status === 'active' && meta.assignee`, return `meta.assignee`
       - Continue on any read error (corrupted meta, missing file)
       - Return null if no match found
       - Export the function for testability

    3. **Wire into syncIssuesHandler** -- After the existing create/update loop processes each feature:
       - After a successful create or update (not skipped, not dry-run), check if the feature now has an issue number
       - Call `findAssigneeForFeature(repoRoot, feature.id)`
       - If assignee found and issue number exists: `await ghExec(['issue', 'edit', String(issueNumber), '--add-assignee', assignee])`
       - Wrap the `--add-assignee` call in try/catch -- on failure, add to `warnings` array: "Failed to set assignee for {featureId}: {error.message}". Do NOT fail the sync.
       - In dry-run mode, skip the assignee sync entirely (no gh calls)
       - Add a small delay after the assignee call (reuse existing `delay(500)` pattern) to avoid rate limiting

    4. **Update tests** in `tests/cli/sync-issues.test.ts`:
       - Add new describe block "assignee propagation"
       - Mock `readdir` and `readMeta` to simulate workstream directories
       - Mock `ghExec` to verify `--add-assignee` calls
       - Test: assignee found and set on created issue
       - Test: assignee found and set on updated issue
       - Test: no assignee found, gh issue edit NOT called
       - Test: assignee sync failure adds warning, does not fail sync
       - Test: dry-run skips assignee sync
       - Test: findAssigneeForFeature unit tests (matching, no match, multiple workstreams, archived skipped)
